# 数据源策略

## 目标

在不同运行环境中优先使用最可靠、最新、最结构化的数据源，同时保持本地脚本兜底能力。不要绑定某个模型或平台当前暴露的具体工具名。

## 优先级

1. 运行环境原生专业数据能力
   - 包括但不限于金融数据库、股票行情、经济数据库、专业数据库、数据源检索、实时行情、分钟线、技术指标、资金流、公告新闻等工具。
   - 模型应自行检查当前可用工具，并选择最适合的工具完成查询。
   - 不要在 skill 中假设工具名称固定。

2. 本地已配置专业数据源
   - 例如 iFinD/Wind/Tushare token、本地 DuckDB/SQLite/CSV 行情库或其他已配置数据源。
   - 只有在能获取结构化、可核验、日期匹配的数据时采用。

3. Skill 自带脚本
   - 当原生工具不可用、返回不完整、无法核验、日期不匹配，或不能提供必要图表时，调用 `skills/qing-stock-analysis/scripts/run_glm_fetch.py`。
   - 该脚本通过 vendored `glmv-stock-analyst` 使用东方财富、AkShare、Tushare、yfinance 等数据源。

## 必须记录

无论数据来自哪里，都要记录或在回答中说明：

- 数据源名称或工具能力描述。
- 查询时间。
- 股票代码和市场。
- 行情数据日期或时间戳。
- 关键字段来源：现价、昨收、开高低收、成交额、K 线周期、技术指标、资金流、新闻公告。
- 缺失字段、冲突字段和可信度说明。

## 采用规则

- 行情、K 线、分钟线、资金流必须来自工具、脚本、网页或本地文件的可核验结果。
- 不能因为模型“看起来知道”行情就直接使用。
- 若两个数据源冲突，优先采用时间戳更新、字段更完整、来源更权威的数据；同时说明冲突。
- 原生工具能提供结构化行情但不能提供图表时，仍需调用本地脚本补 K 线/分时图。
- 原生工具返回的是自然语言摘要时，只能作为辅助信息；关键价格和技术指标仍需结构化数据或可核验来源支持。

## 降级规则

- 当前环境没有原生金融数据工具：直接使用本地脚本。
- 原生工具超时或失败：使用本地脚本。
- 原生工具数据日期不是最近交易日或用户指定日期：使用本地脚本或网页检索补充。
- 本地脚本也缺字段：输出分析降级说明，不编造。
